# Hook Formulas — Catálogo do Slide 1

O Slide 1 é o gate algorítmico do Instagram em 2026: se ele não para o scroll em 1-2 segundos, o resto do carrossel não importa. A taxa de swipe pra slide 3+ define se o post entra na fila de re-distribuição.

Esta referência cataloga as **14 fórmulas de hook** validadas em 2026, com exemplos genéricos multi-nicho prontos pra adaptar.

---

## Regras universais do hook

1. **≤ 10 palavras** no headline principal. Hooks de 6-8 palavras performam melhor que 12+.
2. **Responde duas perguntas** em 1 segundo: "isso é pra mim?" e "o que ganho se eu swipar?"
3. **Stop-scroll visual**: alto contraste, tipografia bold, cor forte. Hook fraco visualmente = hook perdido mesmo com copy boa.
4. **Específico bate genérico**. "5 erros" > "alguns erros". "73%" > "a maioria".
5. **Curiosity gap obrigatório**. O hook deve criar uma pergunta que só o swipe responde — sem entregar o conteúdo já na capa.

---

## Catálogo das 14 fórmulas

Cada fórmula vem com: **uso ideal**, **template de copy** (com placeholders), **3 exemplos genéricos multi-nicho** prontos.

---

### 1. `numerical-list`

**Uso ideal:** framework `listicle`
**Por que funciona:** número promete entrega clara e finita. Cérebro processa "5 X" mais rápido que "alguns X".

**Template:**
```
[NÚMERO] [substantivo plural] [que/para] [benefício ou problema]
```

**Exemplos genéricos:**
- "5 erros que estão te custando seguidores"
- "7 hábitos que separam quem cresce de quem trava"
- "6 ferramentas que mudaram minha rotina"

**Dica:** prefira números ímpares (3, 5, 7) — pesquisas de copy mostram que performam levemente melhor que pares.

---

### 2. `contrarian-reveal`

**Uso ideal:** framework `myth-vs-truth`
**Por que funciona:** promete derrubar crenças do leitor — gera curiosidade defensiva ("será que eu acredito num desses?").

**Template:**
```
[NÚMERO] mitos sobre [tema] que [audiência genérica] ainda acredita
```

**Exemplos genéricos:**
- "3 mitos sobre produtividade que ainda derrubam você"
- "5 verdades incômodas sobre construir uma audiência"
- "4 conselhos que parecem bons mas te travam"

---

### 3. `how-to-numbered`

**Uso ideal:** framework `step-by-step`
**Por que funciona:** "como" sinaliza tutorial; número promete escopo finito.

**Template:**
```
Como [verbo + resultado] em [N] passos [opcional: + qualificador]
```

**Exemplos genéricos:**
- "Como organizar sua semana em 4 passos simples"
- "Como começar algo novo em 3 movimentos"
- "Como dar feedback que muda comportamento em 5 etapas"

---

### 4. `pain-callout`

**Uso ideal:** framework `problem-agitate-solve`
**Por que funciona:** chama direto o leitor que tem o sintoma. Resto ignora — mas quem tem, lê.

**Template:**
```
Se você [sintoma específico reconhecível], [comando/promessa]
```

**Exemplos genéricos:**
- "Se você posta toda semana e ninguém comenta, leia isto"
- "Se você sente que trabalha muito e mostra pouco, este post é seu"
- "Se você adia decisões importantes há semanas, swipe"

---

### 5. `controversial-statement`

**Uso ideal:** framework `contrarian-take`
**Por que funciona:** declaração polêmica gera reação (concordar ou discordar). Algoritmo recompensa o comentário longo.

**Template:**
```
[Marcador "impopular/controverso"]: [afirmação direta contra crença comum]
```

**Exemplos genéricos:**
- "Impopular: postar todo dia está matando sua marca"
- "Polêmico: a maioria dos conselhos sobre disciplina é mentira"
- "Vou ser direto: networking não funciona do jeito que te ensinaram"

---

### 6. `methodology-naming`

**Uso ideal:** framework `framework-reveal`
**Por que funciona:** nomear um método cria propriedade intelectual. Cada uso futuro do nome cita o autor.

**Template:**
```
O método [NOME CURTO/SIGLA] para [resultado claro]
```

**Exemplos genéricos:**
- "O método 3-2-1 para tomar decisões difíceis"
- "A regra dos 5 minutos para começar qualquer coisa"
- "O framework C.L.A.R.O para falar em público"

**Dica:** sigla memorável (3-5 letras) ou número-com-ritmo ("3-2-1", "80-20") funciona melhor que nome conceitual longo.

---

### 7. `mistake-call-out`

**Uso ideal:** framework `mistakes-to-avoid`
**Por que funciona:** promete listar erros que o próprio leitor pode estar cometendo. Curiosidade ansiosa = stop-scroll forte.

**Template:**
```
[N] erros que [audiência] [comete sem perceber / faz toda semana / repete há anos]
```

**Exemplos genéricos:**
- "5 erros que iniciantes cometem sem perceber"
- "7 hábitos que sabotam seus resultados toda semana"
- "4 frases que você diz e não deveria"

---

### 8. `transformation-tease`

**Uso ideal:** framework `case-study-narrative`
**Por que funciona:** promete narrativa de transformação — gancho clássico de storytelling.

**Template:**
```
Como [sujeito genérico] foi de [estado X] para [estado Y] em [tempo/condição]
```

**Exemplos genéricos:**
- "Como uma pessoa comum saiu do zero e construiu autoridade em 6 meses"
- "De refém da rotina para dono do tempo — uma transformação real"
- "Como alguém saiu de invisível a referência em sua área"

**Importante multi-nicho:** "pessoa", "alguém", "profissional" — nunca uma profissão específica.

---

### 9. `before-after-tease`

**Uso ideal:** framework `before-after-transformation`
**Por que funciona:** promete comparação visual; cérebro processa contraste instantaneamente.

**Template:**
```
De [estado inicial] para [estado final] — [sem/com qualificador]
```

**Exemplos genéricos:**
- "De caos a controle — sem ferramentas pagas"
- "De invisível a procurado em 90 dias"
- "Antes vs depois de mudar UMA coisa na rotina"

---

### 10. `compendium-promise`

**Uso ideal:** framework `cheat-sheet`
**Por que funciona:** promete entrega total e referencial. Frame natural de "salvar pra sempre".

**Template:**
```
Tudo que você precisa saber sobre [tema] em [N] slide(s) / 1 post
```

**Exemplos genéricos:**
- "Tudo sobre escrever bem em 1 post"
- "Cheat sheet completo de produtividade — salve para consultar"
- "O guia visual definitivo de comunicação clara"

---

### 11. `stat-shocker`

**Uso ideal:** framework `data-driven-insight`
**Por que funciona:** número alto + comportamento inesperado quebra o piloto automático do feed.

**Template:**
```
[N%] das pessoas [comportamento contra-intuitivo / problema crônico]
```

**Exemplos genéricos:**
- "73% das pessoas abandonam metas no primeiro mês — e o motivo te surpreende"
- "92% das ideias morrem por causa de UM erro evitável"
- "8 em cada 10 profissionais sabotam o próprio crescimento sem saber"

**Cuidado:** o stat tem que ser **verificável ou claramente arredondado**. Stats fabricados destroem credibilidade no review humano.

---

### 12. `confession`

**Uso ideal:** framework `vulnerable-story`
**Por que funciona:** vulnerabilidade pega audiência saturada de autoridade performada. Sinaliza humano real.

**Template:**
```
Eu [errei / fracassei / acreditei] em [coisa específica] por [tempo] — aqui está o que aprendi
```

**Exemplos genéricos:**
- "Eu acreditei nessa mentira por anos — e quase desisti por causa dela"
- "Cometi o mesmo erro 5 vezes antes de entender o óbvio"
- "Por muito tempo segui o conselho errado — corrigi e tudo mudou"

---

### 13. `peek-tease`

**Uso ideal:** framework `behind-the-scenes`
**Por que funciona:** promete acesso a algo normalmente escondido. Frame de exclusividade.

**Template:**
```
O que [ninguém te conta / ninguém mostra] sobre [processo / rotina]
```

**Exemplos genéricos:**
- "O que ninguém te conta sobre construir audiência"
- "Por dentro do processo que parece simples (mas não é)"
- "A parte invisível de qualquer projeto que dá certo"

---

### 14. `question-pile`

**Uso ideal:** framework `mini-faq`
**Por que funciona:** sinaliza autoridade ("recebo muitas perguntas sobre X") + promete entrega densa.

**Template:**
```
As [N] perguntas que mais [recebo / escuto] sobre [tema]
```

**Exemplos genéricos:**
- "As 5 perguntas que mais recebo sobre começar do zero"
- "6 perguntas que todo mundo faz sobre produtividade — e as respostas honestas"
- "As 4 dúvidas mais comuns sobre construir presença online"

## Mapeamento rápido framework → hook formula

| Framework | Hook fórmula default |
|-----------|---------------------|
| listicle | `numerical-list` |
| myth-vs-truth | `contrarian-reveal` |
| step-by-step | `how-to-numbered` |
| problem-agitate-solve | `pain-callout` |
| contrarian-take | `controversial-statement` |
| framework-reveal | `methodology-naming` |
| mistakes-to-avoid | `mistake-call-out` |
| case-study-narrative | `transformation-tease` |
| before-after-transformation | `before-after-tease` |
| cheat-sheet | `compendium-promise` |
| data-driven-insight | `stat-shocker` |
| vulnerable-story | `confession` |
| behind-the-scenes | `peek-tease` |
| mini-faq | `question-pile` |

A skill pode usar **outra fórmula** se houver razão clara (ex: framework `listicle` com tom contrarian → `controversial-statement`). Mas a fórmula default é a aposta segura.

---

## Anti-padrões — hooks que NÃO usar

1. **Vague/abstrato:** "Reflexões sobre liderança" — sem stop-scroll, sem promessa.
2. **Pergunta retórica sem tensão:** "Você já pensou em ser melhor?" — cérebro responde "não" e segue.
3. **Auto-elogio:** "5 coisas que aprendi sendo CEO" — soa egótico, baixo CTR fora de audiência fiel.
4. **Bait sem entrega:** "Você não vai acreditar no que aconteceu" — algoritmo penaliza clickbait sem substância.
5. **Hashtag/emoji excessivo na copy do hook:** ruído visual mata clareza.

---

## Para o template multi-nicho

Como o template é genérico e o **cliente final substitui a copy do hook depois**, a copy de exemplo do Slide 1 que vai no template precisa:

- Seguir a **fórmula** correta (essa fica como contrato visual/estrutural).
- Ser **genérica o suficiente** pra qualquer nicho ver claramente o padrão e adaptar.
- Caber no `data-template-element` com `max-chars` definido pelo designer.

A skill `gp2-template-suggester` instrui o designer a usar **uma das fórmulas acima** + uma das copys de exemplo. O cliente final, depois, substitui mantendo a fórmula.
